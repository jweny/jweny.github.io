sqli & xss

sqli & xss

一、sqli

1.1 原理与危害

sql注入是将恶意的sql查询或者语句添加到应用的输入参数中，然后在后台的服务器数据库上解析执行进行攻击，如果用户输入恶意sql查询代码，web应用又没有对输入的sql查询进行审查，就会出现该漏洞。危害：

  1. 猜解后台数据库，这是最常见的情况，窃取大量数据库敏感数据

  2. 借助数据库进行提权

  3. 绕过验证直接登录

1.2 手工判断注入点

**关键点：让查询条件闭合，以下均为简单示例，实际场景中除了闭合引号，还可能存在括号等 **

  • 初步探测：

    • 加入单引号 ‘ 提交，如果报错，则该网站可能就存在注入漏洞。

    • 加入双引号 “ 提交，如果报错，则该网站可能就存在注入漏洞。

    • 如果单引号报错，双引号不报错；或者单引号不报错，双引号报错，说明存在注入漏洞。

  • 数字型sql注入探测：

    select user from users where id = 1

    • 方法1：尝试 news.asp?id=123 and 1=1 和 news.asp?id=123 and 1=2 返回不同的页面,说明存在注入漏洞。（分析：and前后语句有一个为假则整个查询条件为假，因此and 1=1 返回正常页面，and 1=2 时会查询错误或不返回记录。）

    • 方法2：尝试 news.asp?id=123 or 2>1 和 news.asp?id=123 or 1>2 返回不同的页面,说明存在注入漏洞。（分析：or前后语句有一个为真则整个查询条件为真，or 1=1 永真条件返回所有记录，or 1=2 返回原来相同的结果）

    • 方法3：尝试对注入点进行 -1 和 -0。例如 news.asp?id=123-0 和 news.asp?id=123-1 如果返回不同的页面，说明存在注入漏洞。（分析：123-0时返回原来相同的结果，如果123-1时返回其他页面或页面没找到，表示存在注入漏洞。）

  • 字符型sql注入探测：

    select user from users where id = '1'

    • 方法1：news.asp?id=123’ and ‘1’=’1 and news.asp?id=123’ and ‘1’=’2 分别返回不同的页面,说明存在注入漏洞。（分析：加入’ and ‘1’=’1返回正确页面，and ‘1’=’2 返回查询记录或不返回记录，说明有注入漏洞。）

    • 或将上述语句’ 换成双引号

    • 其他方法可类比数字型，将数字型转为字符型

  • 表单提交/搜索框时，还可以通过通配符判断。例如先搜索 ‘，如果出错，然后搜索%，如果正常返回，说明存在注入漏洞。

  • 报错形式：

    • 数据库查询错误直接回显在页面上
    • 错误隐藏在页面源代码中
    • 检测到错误跳转到另一个页面
    • 返回HTTP错误代码500或重定向302
    • 适当处理错误结果，显示一个通用错误页面

1.3 使用sqlmap

1.3.1 中文手册

https://parrotsec-cn.org/t/topic/834

1.3.2 sqlmap探测分类

  • B: Boolean-based blind SQL injection（布尔型盲注）

    通过判断页面返回情况获得想要的信息，比如数据库版本。
    id=1’ and select(substring(version(),1,1))=5 #
    id=1’ and length(database())>1 #

  • E: Error-based SQL injection（报错型注入）

    id=1’ and/**/extractvalue(1,concat(char(126),md5(1274754846)))and’
    当然如果报错的数据不能回显，就无法使用报错注入，这时候我们就可以考虑是否存在盲注。

  • U: UNION query SQL injection（联合查询注入）

    http://hello.com/view?id=1 UNION ALL SELECT SCHEMA_NAME, DEFAULT_CHARACTER_SET_NAME FROM INFORMATION_SCHEMA.SCHEMATA

  • S: Stacked queries SQL injection（可多语句查询注入）

    http://hello.com/view?id=1;update t1 set content = ‘aaaaaaaaa’

  • T: Time-based blind SQL injection（时间型盲注）

    http://hello.com/view?q=abc' AND (SELECT * FROM (SELECT(SLEEP(5)))VCVe) OR 1 = ‘
    返回时间应该 >= 5s，要考虑网络延迟。

二、xss

2.1 漏洞原理

往Web页面里插入恶意javaScript代码，当用户浏览该页之时，嵌入其中Web里面的javaScript代码会被执行，从而达到恶意攻击用户的目的。

2.2 检测原理

  • 发送随机字符
  • 确认参数回显
  • 通过html/js解析确认回显位置
  • 根据回显的上下文发送不同的payload进行测试
  • 使用html，js语法树解析是否多出来标签，属性，js语句等来确定是否执行成功

2.3 注意事项

  • payload可能有多种原因无法直接回显（弹窗），所以建议去查看响应的DOM树（F12开发者模式-elements），看是否多出来标签/属性/js语句。
  • chrome 内置 CSP，因此想直接回显的话建议使用firefox。
